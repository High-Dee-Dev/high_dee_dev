Getting Started

Welcome to the LuxeThreads editing guide! This section will help you understand the template structure and how to begin customizing it for your fashion brand.

1 Understand the File Structure

Before making changes, familiarize yourself with the template's file structure:

fashion-ecommerce-template/

The main folders are:

  • root folder - Contains the main HTML files
  • pages/ - All additional HTML pages
  • assets/css/ - Stylesheets
  • assets/js/ - JavaScript files
  • assets/images/ - All images organized by type

2 Choose Your Editor

Use a code editor that you're comfortable with. Recommended editors:

  • Visual Studio Code (Free, recommended)
  • Sublime Text
  • Atom
  • Notepad++ (Windows)

Install the editor and open the entire template folder in it.

3 View the Template

Open index.html in your web browser to see how the template looks. Navigate through all pages to understand the complete structure.

Tip: Keep the browser open while editing. After saving changes in your editor, refresh the browser (F5) to see the updates.

4 Make a Backup

Before making any changes, create a backup copy of the entire template folder. This way, you can always revert to the original if needed.

Important: Always work on a copy, not the original files, especially if you plan to make extensive customizations.

Changing Colors

The template uses CSS variables for colors, making it easy to change the entire color scheme in one place.

1 Locate Color Variables

Open assets/css/style.css and find the :root selector at the top of the file:

/* ===== CSS Variables ===== */
:root {
  /* Primary Colors */
  --primary-dark: #1a1a1a;
  --primary-light: #f8f9fa;
  --accent-color: #d4af73;
  --accent-hover: #c19a5b;

  /* Text Colors */
  --text-dark: #212529;
  --text-muted: #6c757d;
  --text-light: #f8f9fa;
}

2 Change Color Values

Replace the hex color codes with your own brand colors. Here's what each variable controls:

Variable Default Color Used For
--primary-dark #1a1a1a Dark backgrounds, footer, buttons
--primary-light #f8f9fa Light backgrounds, cards
--accent-color #d4af73 Primary accent, highlights, hover states
--accent-hover #c19a5b Accent color on hover
--text-dark #212529 Main text color

3 Test Your Changes

After changing the colors, save the file and refresh your browser. The entire template should update with your new color scheme.

Tip: Use a color palette generator like Coolors or ColorHunt to create harmonious color schemes for your brand.

4 Additional Color Customization

If you need to change specific elements that aren't controlled by the variables, you can add custom CSS at the end of the style.css file:

/* Custom color overrides */
.navbar {
  background-color: your-color !important;
}

.btn-primary {
  background-color: your-color;
  border-color: your-color;
}

Replacing Images

All demo images should be replaced with your own product and brand images before launching your website.

1 Image Folder Structure

The images are organized in subfolders within assets/images/:

  • products/ - Product photos
  • banners/ - Hero and promotional banners
  • blog/ - Blog post images
  • testimonials/ - Customer photos
  • gallery/ - Instagram-style gallery
  • brands/ - Brand logos
  • icons/ - SVG icons and badges

2 Method 1: Replace with Same Names

The easiest way is to replace the existing images with your own images while keeping the same file names:

  1. Take your product photo
  2. Rename it to match an existing file (e.g., product1.jpg)
  3. Replace the file in the appropriate folder
  4. Refresh your browser to see the change

Image Sizing Tips:

  • Product images: 800x1000px (portrait orientation)
  • Banner images: 1920x800px (landscape orientation)
  • Blog images: 800x500px
  • Optimize all images for web (use tools like TinyPNG)

3 Method 2: Update Image Paths

If you want to use different file names, update the image paths in the HTML files:

<!-- Find this in index.html -->
<img src="assets/images/products/product1.jpg" alt="Product name">

<!-- Change to your image path -->
<img src="assets/images/products/my-product.jpg" alt="My Product Name">

Search for all instances of the old file name and replace them with your new file name.

4 Image Optimization

For best performance, optimize your images before uploading:

  • Use JPEG format for photos
  • Use PNG format for logos and graphics with transparency
  • Use SVG for icons and simple graphics
  • Compress images using:
    • TinyPNG (online)
    • ImageOptim (Mac)
    • RIOT (Windows)
  • Add descriptive alt text for SEO and accessibility

Editing Text Content

Replace all demo text with your own brand messaging, product descriptions, and content.

1 Finding Text to Edit

Open any HTML file in your editor. Text content is found between HTML tags:

<!-- Page title -->
<h1 class="display-4">Summer Collection </h1>

<!-- Product name -->
<h2 class="product-title">Elegant Silk Evening Dress</h2>

<!-- Product description -->
<p class="product-description">
  Crafted from premium mulberry silk, this elegant evening dress...
</p>

Simply replace the text between the opening and closing tags with your own content.

2 Text Editing Best Practices

  • Keep it concise: Online readers scan content
  • Use headings: Break content with H1, H2, H3 tags
  • Add keywords: Include relevant keywords for SEO
  • Be authentic: Write in your brand voice
  • Check spelling: Use a spell checker
  • Mobile-friendly: Short paragraphs work better on mobile

3 Common Text Sections to Update

Section File What to Update
Navigation All HTML files Menu items, brand name
Hero Banner index.html Headline, subheadline, button text
Product Info shop.html, product-details.html Product names, descriptions, prices
About Page about.html Brand story, mission, team bios
Footer All HTML files Contact info, copyright, links

4 Using Find and Replace

To update text that appears multiple times (like brand name), use your editor's Find and Replace feature:

  1. Open the Find dialog (usually Ctrl+F or Cmd+F)
  2. Enter the text to find (e.g., "LuxeThreads")
  3. Enter the replacement text (e.g., "Your Brand Name")
  4. Use "Replace All" to update all instances

Caution: Be careful with "Replace All" - make sure you're only replacing the specific text you intend to change. Preview changes before applying to all files.

Modifying Products

The template includes sample product data that you need to replace with your actual products.

1 Understanding Product Data Structure

Product information is stored in two places:

  1. HTML files: For static display
  2. JavaScript files: For dynamic functionality (cart, filtering, etc.)

Open assets/js/main.js and find the products array:

// Sample product data
const products = [
  {
    id: 1,
    name: "Elegant Silk Evening Dress",
    category: "Dresses",
    price: 189.99,
    oldPrice: 249.99,
    image: "assets/images/products/product1.jpg",
    rating: 4.5,
    reviewCount: 128,
    sale: true,
    new: false
  },
  // ... more products
];

2 Adding Your Products

Replace the sample products with your own product data:

  1. Copy the product object structure
  2. Update all values with your product information
  3. Add or remove products as needed
  4. Ensure each product has a unique ID
  5. Update image paths to your product photos

Product Data Fields:

  • id: Unique number for each product
  • name: Product title (keep it descriptive)
  • category: Product category for filtering
  • price: Current price (number)
  • oldPrice: Original price for sales (null if not on sale)
  • image: Path to product image
  • rating: Average rating (1-5)
  • reviewCount: Number of reviews
  • sale: true/false for sale badge
  • new: true/false for new badge

3 Updating Product HTML

For static product displays (like featured products), update the HTML directly:

<!-- Find product cards in HTML files -->
<div class="product-card">
  <img src="assets/images/products/product1.jpg" alt="Elegant Silk Dress">
  <h3>Elegant Silk Evening Dress</h3>
  <p class="price">$189.99</p>
</div>

Update the image source, alt text, product name, and price with your product details.

4 Connecting to a Real Database

For a production website with many products, you'll want to connect to a backend database:

  1. Set up a server (Node.js, PHP, Python, etc.)
  2. Create a database to store products
  3. Build API endpoints to serve product data
  4. Update JavaScript to fetch products from your API
  5. Implement search, filtering, and pagination on the server

For Small Stores: If you have fewer than 50 products, you can manage them directly in the JavaScript file. For larger stores, a database is essential.

Changing Fonts

The template uses Google Fonts for typography. You can easily change to different fonts that match your brand personality.

1 Current Font Setup

Open any HTML file and look in the <head> section:

<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:wght@400;500;600;700&family=Montserrat:wght@300;400;500;600;700&display=swap" rel="stylesheet">

The template uses two fonts:

  • Playfair Display: For headings (elegant, serif font)
  • Montserrat: For body text (clean, sans-serif font)

2 Choosing New Fonts

Visit Google Fonts to browse available fonts:

  1. Search for fonts that match your brand
  2. Select 1-2 fonts (one for headings, one for body)
  3. Click "Select this style" for each weight you need
  4. Copy the link code from the "Use on the web" section

Font Pairing Tips:

  • Pair a serif with a sans-serif for contrast
  • Don't use more than 2-3 different fonts
  • Ensure good readability, especially for body text
  • Consider loading only the weights you actually use

3 Updating Font Links

Replace the existing Google Fonts link with your new one in all HTML files:

<!-- Replace this line in all HTML files -->
<link href="https://fonts.googleapis.com/css2?family=YOUR_NEW_FONTS&display=swap" rel="stylesheet">

Use Find and Replace to update all HTML files at once.

4 Updating CSS Font Variables

Open assets/css/style.css and update the font variables:

/* Typography */
:root {
  --font-heading: 'Your Heading Font', serif;
  --font-body: 'Your Body Font', sans-serif;
}

Replace 'Your Heading Font' and 'Your Body Font' with the actual font family names from Google Fonts.

Adding New Pages

Learn how to create additional pages for your website while maintaining consistent design.

1 Duplicate an Existing Page

The easiest way to create a new page is to duplicate an existing one:

  1. Copy an HTML file that has a similar layout to what you need
  2. Rename the copy (e.g., new-page.html)
  3. Place it in the pages/ folder
  4. Open the new file in your editor

Good Templates to Duplicate:

  • about.html - For content pages
  • shop.html - For product listing pages
  • contact.html - For forms and information

2 Update Page Content

Edit the duplicated page with your new content:

  1. Update the <title> in the head section
  2. Change the main heading and page content
  3. Update the breadcrumb navigation if present
  4. Remove or modify sections as needed
<!-- Update page title -->
<title>Your New Page Title | Your Brand</title>

<!-- Update main heading -->
<h1 class="display-5">Your New Page Heading</h1>

3 Add to Navigation

To make your new page accessible, add it to the main navigation:

  1. Open index.html and all page files in the pages/ folder
  2. Find the navigation section (look for <nav class="navbar">)
  3. Add a new menu item in the appropriate place
<!-- Add this to the navigation -->
<li class="nav-item">
  <a class="nav-link" href="pages/new-page.html">
    New Page
  </a>
</li>

4 Test and Link

After creating your new page:

  1. Open it in your browser to check the layout
  2. Test all links on the page
  3. Ensure the navigation highlights correctly
  4. Add links to the new page from other pages if needed
  5. Test on mobile devices

Success Tip: Keep a consistent layout and design across all pages to maintain a professional look and user experience.

Customizing Animations

The template includes subtle, professional animations that can be easily customized or disabled.

1 Animation Files

All animations are defined in assets/css/animations.css. This file contains:

  • CSS keyframe animations
  • Animation utility classes
  • Hover effects and transitions
  • Scroll-based animations

The main stylesheet (style.css) includes animation classes for specific elements.

2 Disabling Animations

If you prefer fewer animations (for performance or preference):

/* Disable all animations */
*,
*::before,
*::after {
  animation-duration: 0.01ms !important;
  animation-iteration-count: 1 !important;
  transition-duration: 0.01ms !important;
}

Add this CSS at the end of animations.css to disable all animations.

3 Modifying Specific Animations

To change a specific animation, find its keyframe definition:

/* Example: Modify fadeIn animation */
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

Adjust the values to change the animation behavior. For example, change translateY(20px) to translateY(10px) for a smaller movement.

4 Changing Animation Timing

Animation timing is controlled by the animation property:

.animate-fade-in {
  animation: fadeIn 0.6s ease-out;
}

To make animations faster or slower, change the duration (0.6s). To change the timing function, replace ease-out with another value like linear, ease-in, or cubic-bezier().

Timing Functions:

  • linear - Constant speed
  • ease - Slow start and end, fast middle (default)
  • ease-in - Slow start
  • ease-out - Slow end
  • ease-in-out - Slow start and end
  • cubic-bezier() - Custom timing

Connecting to a Backend

While the template works as a static site, for a full e-commerce store you'll need to connect it to a backend system.

1 Backend Options

Choose a backend technology based on your needs and skills:

Technology Best For Difficulty
Node.js + Express JavaScript full-stack Intermediate
PHP + MySQL Traditional web hosting Beginner to Intermediate
Python + Django Rapid development Intermediate
Ruby on Rails Convention over configuration Intermediate
Headless CMS Content management without coding Beginner

2 Headless CMS Option

For beginners, a headless CMS is the easiest way to add backend functionality:

  1. Sign up for a service like:
    • Strapi (open source)
    • Contentful
    • Sanity
    • Prismic
  2. Create content models for Products, Categories, etc.
  3. Add your products through the CMS admin panel
  4. Use the CMS API to fetch data in your template

Benefits of Headless CMS:

  • No server setup required
  • Easy content management
  • Scalable and secure
  • API-first architecture

3 Connecting with JavaScript

To fetch data from your backend API, update the JavaScript files:

// Replace static product data with API call
async function loadProducts() {
  try {
    const response = await fetch('https://your-api.com/products');
    const products = await response.json();
    displayProducts(products);
  } catch (error) {
    console.error('Error loading products:', error);
  }
}

Update all data fetching in main.js, cart.js, and product.js to use your API endpoints.

4 E-Commerce Platforms

For a complete solution, consider integrating with an e-commerce platform:

  • Shopify: Use Storefront API
  • WooCommerce: WordPress plugin with REST API
  • BigCommerce: Headless commerce platform
  • Saleor: Open-source GraphQL API

These platforms handle products, inventory, payments, and shipping, allowing you to focus on the frontend design.

Implementation Path: Start with static data → Move to headless CMS → Integrate with e-commerce platform as your business grows.
